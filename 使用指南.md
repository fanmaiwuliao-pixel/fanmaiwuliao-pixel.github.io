# 个人网站使用指南

网站结构参考 historio.asia，包含：引得（首页）、学术简历、所有文章。全程免费，无广告，不需要写代码。

## 一、首次部署（约 10 分钟，只做一次）

1. 注册/登录 [github.com](https://github.com)。
2. 点右上角 **+** → **New repository**：
   - 仓库名填 `你的用户名.github.io`（这样网址最简洁），设为 **Public**，点 **Create repository**。
3. 在新仓库页面点 **uploading an existing file**（或 Add file → Upload files），把本压缩包里 `mysite` 文件夹**内的所有内容**（不含 mysite 文件夹本身）拖进去，点 **Commit changes**。
   - 注意：`_posts`、`_layouts` 等文件夹要保持原有结构。网页上传不便保留文件夹时，可用 GitHub Desktop，或上传 zip 后联系我帮你处理。
4. 进仓库的 **Settings → Pages**，Source 选 **Deploy from a branch**，Branch 选 `main` / `(root)`，保存。
5. 等 1–2 分钟，访问 `https://你的用户名.github.io` 即可看到网站。

## 二、日常「上传东西」

全部在 GitHub 网页上操作，改完 Commit，1–2 分钟后网站自动更新。

**发新文章**：进入 `_posts` 文件夹 → Add file → Create new file → 文件名 `2026-08-01-biaoti.md`（日期-拼音标题）→ 开头写：

```
---
title: 文章标题
categories: [日记]
---
```

下面接正文（Markdown 格式），Commit 即可。

**上传 PDF / 图片**：进入 `files` 文件夹 → Add file → Upload files。之后在文章里引用：

- PDF 下载链接：`[下载 PDF](/files/文件名.pdf)`
- 插图：`![说明](/files/图片名.jpg)`

**修改简历**：打开 `cv.md`，点铅笔图标编辑，Commit。

**修改首页介绍**：同样编辑 `index.md`。

## 三、初始必改的占位内容

- `_config.yml`：网站名称、副标题、作者、Email
- `index.md`：首页自我介绍
- `cv.md`：学历、经历、联系方式
- `_posts/2026-07-05-diary-sample.md`：示例日记，可删除

## 四、以后想要更多

自定义域名（如 historio.asia 那样）可在 Settings → Pages 绑定自购域名；想加「论著作品」「研究主题」等页面，仿照 `cv.md` 新建一个 `.md` 文件并在 `_layouts/default.html` 的导航里加一行链接即可——也可以随时找我帮你加。
